java注解

注解可以被编译器打包进入class文件，是一种用作标注的元数据。

  • 由编译器使用的注解@Override让编译器检查该方法是否正确地实现了覆写。这类注解不会被编译进入.class文件，在编译后就被扔掉了
  • 由工具类处理.class文件使用的注解。
  • 程序运行期能够读取的注解，加载后一直存在于JVM中，例如@PostConstruct方法在调用构造方法后自动被调用。
